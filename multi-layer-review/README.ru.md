# multi-layer-review

[🇬🇧 Read in English](README.md)

Скилл для Claude Code, который прогоняет спек или ТЗ через до 5 **независимых слепых ревьюеров** параллельно — до начала разработки. Каждый ревьюер работает независимо: никакой передачи находок, никакого усиления паники. Совпадение находок от разных ревьюеров — сильный сигнал.

---

## Как это работает

**Шаг 0.5 — Получить спек.** Файл, путь или текст.

**Шаг 0.6 — Review Brief (автоматически).** Формирует контекст: что за проект, что за документ, есть ли уже код по этой теме, связанные файлы. Передаётся всем ревьюерам.

**Шаг 0.7 — Ambiguity Gate (автоматически).** Проверяет спек на размытые требования, противоречия, отсутствующие ограничения. Если находит — спрашивает уточнить или продолжить.

**Шаг 0.8 — Project Context Snapshot (автоматически).** Одноразовый обход проекта: структура, манифесты, релевантный код, конфиги. Передаётся всем ревьюерам, чтобы анализ был основан на реальном состоянии проекта, а не на абстрактных рассуждениях.

**Шаги 1–5 — Слепые ревьюеры (параллельно).** Каждый проверяет спек независимо с брифом + контекстом проекта. Ни один ревьюер не видит находок другого. Они используют Read/Glob/Grep для проверки реальных файлов проекта.

**Финал — Синтез.** Все находки объединяются, дедуплицируются и сортируются по приоритету. Независимое совпадение (2+ ревьюера нашли одно и то же) повышает severity на один уровень.

## Ревьюеры

| # | Роль | Что ищет |
|---|------|----------|
| 1 | **Requirements Traceability** — аналитик | Правильную ли задачу решаем? Соответствует ли решение бизнес-цели? Есть ли критерии успеха? |
| 2 | **Plan Agent** — архитектор | Структурные пробелы, качество API, граничные случаи, противоречия, совместимость с существующей архитектурой |
| 3 | **Codex CLI** — разработчик *(требует [Codex CLI](https://github.com/openai/codex))* | Риски реализации, небезопасные паттерны, проблемы зависимостей, отсутствующая обработка ошибок |
| 4 | **Devil's Advocate** — скептичный пользователь | "Что если..." — UX-провалы, неожиданное поведение, что сломается в первый же день |
| 5 | **Robustness Checklist** — системный инженер | Оценка 0–3 по 8 критериям: error handling, concurrency, безопасность, тестируемость и др. |

Ревьюеров можно выбирать — не обязательно запускать всех пятерых.

**Формат вывода:** Каждая находка — `[Critical/Important/Minor] Section: "X" — описание`. Максимум 10 находок на ревьюера. "No issues found" — валидный результат.

## Что нового в v3.0

- **Слепые ревью** — больше никакой передачи находок между ревьюерами (устраняет confirmation bias и эскалацию паники)
- **Параллельный запуск** — ревьюеры работают одновременно, быстрее результат
- **Review Brief** — автоматический контекст проекта/документа для всех ревьюеров
- **Project Context Snapshot** — одноразовый обход кодовой базы, общий для всех
- **Анализ на основе фактов** — ревьюеры используют Read/Glob/Grep для проверки реальных файлов
- **Единый формат вывода** — единообразные severity, ссылки на секции, максимум 10 находок
- **Без выдуманных проблем** — явное правило против общих советов и гипотетических проблем

---

## Когда использовать

- Написал спек или ТЗ — хочешь найти проблемы до начала разработки
- Проектируешь API или архитектуру модуля
- Не уверен что правильно понял задачу или решаешь ли ту проблему

---

## Пример

```
Ты: проверь ТЗ
```

Скилл спросит каких ревьюеров запустить, попросит спек (файлом или текстом), соберёт контекст проекта, запустит ревьюеров параллельно и выдаст синтез с указанием кто что нашёл. Результат сохраняется в файл.

## Установка

Посмотри [что устанавливается](SKILL.md) перед запуском команды.

**macOS / Linux:**
```bash
mkdir -p ~/.claude/skills/multi-layer-review
curl --fail -o ~/.claude/skills/multi-layer-review/SKILL.md \
  https://raw.githubusercontent.com/GuruDevvv/my-claude-skills/main/multi-layer-review/SKILL.md
```

**Windows (Git Bash или WSL):**
```bash
mkdir -p "$USERPROFILE/.claude/skills/multi-layer-review"
curl --fail -o "$USERPROFILE/.claude/skills/multi-layer-review/SKILL.md" \
  https://raw.githubusercontent.com/GuruDevvv/my-claude-skills/main/multi-layer-review/SKILL.md
```

Перезапусти сессию Claude Code — терминал с `claude`, или перезагрузи окно VS Code.

**Проверка:** напиши агенту `multi-layer review` — скилл активируется автоматически.
