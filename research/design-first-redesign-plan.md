# План редизайна design-first v3

Цель: сдвиг от **«генератор корректных вариантов»** к **«арт-директор, который заставляет себя рисковать»**. Источник решений — `design-first-wow-research.md`.

Подход: **этот план → тест на 1-2 прототипах → правка плана при необходимости → финальный SKILL.md**.

---

## Принцип, который пронизывает весь скилл

> Технически корректный ≠ запоминающийся. Каждый структурный прототип ведёт ОДНА смелая визуальная идея (концепция), из которой выводится всё остальное — цвет, шрифт, layout, мотион, изображения. Без концепции получается «6 скинов одной страницы».

---

## Структура шагов (что меняется)

### Step 1 — Context Interview — *оставить почти как есть*
Добавить 1 вопрос: **«Anti-slop / смелость»** — «Насколько смелым может быть дизайн? (от "консервативно, это банк" до "удиви меня, не бойся")». Это калибрует уровень риска.

### Step 2 — Reference Analysis — *оставить, усилить*
- Оставить Design DNA таблицу (это лучшая часть).
- Добавить: **смотреть рефы ВНЕ категории продукта** — арт-движения, печатный дизайн, мода, упаковка, кино. Источник настоящего сюрприза.

### Step 2.5 — Design Brief — *расширить новыми полями*
Добавить в `BRIEF.md`:
- **Концепция на прототип** (concept-first, не «warm tones», а «телеграм из Берлина 1920-х»). Колонка в таблице плана становится главной.
- **Negative rules** (что ЗАПРЕЩЕНО): no purple-on-white, no centered 3-col grid, no pill buttons, no Inter/Roboto — + проектные анти-рефы.
- **«Скучная версия»** — поле «как выглядел бы банальный вариант? → избегать его».
- **Image plan** — какой Tier изображений используем (0/0b/1/2/3), какие картинки нужны (hero, scroll, секции).

### Step 3 — Generate Prototypes — *ЯДРО редизайна*

**3a. Art Direction слой (НОВОЕ, главное).**
Перед генерацией каждого структурного прототипа — одностроковая концепция + выведенные из неё: цвет (из мудборда/концепта, НЕ из каталога), шрифт (отличительный), сигнатурный ход.

**3b. Встроить Anthropic `<frontend_aesthetics>` блок** (дословно) в промпт каждого прототипа. Это официальный анти-slop артефакт.

**3c. Типографика — запрет + мандат + кириллический гейт.**
- НИКОГДА: Inter, Roboto, Arial, system, Open Sans, Lato.
- Брать отличительные: Fraunces, Playfair Display, Clash Display, Bricolage Grotesque, Cormorant, IBM Plex, Unbounded и т.д.
- Вес-экстремумы (200 vs 800, не 400 vs 600), скачки размера 3x+.

**ЖЁСТКИЙ ГЕЙТ ДЛЯ RU (и любого кириллического контента) — правило пользователя, подтверждено тестом:**
Многие «отличительные» шрифты из американских подборок **НЕ имеют кириллицы** и для русского НЕПРИГОДНЫ. Проверено на тесте Сновидца: Fraunces, Clash Display, Bricolage Grotesque, Cabinet Grotesque, Satoshi, DM Serif, Space Grotesk — **без кириллицы**, дадут □□□ или латиничный фолбэк.
- Перед выбором шрифта для RU — **обязательно убедиться, что он поддерживает Cyrillic subset** (fonts.google.com → вкладка Languages → Cyrillic; или `&subset=cyrillic` / наличие в проверенной таблице ниже).
- Это HARD-гейт в WOW-чеклисте: «RU-контент → каждый шрифт проверен на кириллицу?». Сломанная кириллица мгновенно убивает прототип.
- Отличительные И кириллица-safe (проверенные): **Cormorant Garamond, Playfair Display, Literata, PT Serif, Merriweather, Unbounded, Manrope, Oswald, Philosopher, Podkova, Marmelad, Caveat, Jost, Commissioner, Bitter, Noto Serif**.
- Для латиницы — ограничений нет, весь арсенал ресёрча доступен. → в скилле явная развилка **RU vs Latin**: разные пулы отличительных шрифтов.
- Обновить **кириллическую таблицу**: добавить колонку «отличительность», явно пометить «латиничные-only» как запрещённые для RU.

**3d. Цвет из концепции.** Поднять текущее предупреждение (строка 430 старого скилла) в core-принцип ДО генерации. Палитра выводится из мудборда/концепта. ui-ux-pro-max понижается: «опциональный референс для перебора стилей», НЕ «foundation».

**3e. Изображения (НОВОЕ) — tiered-стратегия. ИСПРАВЛЕНО ПО ТЕСТУ.**
- ⚠️ **Pollinations (Tier 0b) ВЫЧЕРКНУТ** — без ключа отдаёт `402 Queue full`, картинки не грузятся. Ресёрч ошибался.
- **Tier 0** (бесплатно, надёжно): **Lorem Picsum** `picsum.photos/seed/.../1600/900` — подтверждено 200/JPEG. Минус: не по ключевику (держать curated ID или принять случайность).
- **Tier 2** (лучший, дефолт когда доступен): **Higgsfield MCP `generate_image`** — bespoke под концепцию, ~2 кредита/картинку, скачать локально (`assets/`). Проверено на тесте: nano_banana_pro дал идеально-концептуальные фото за 4 кредита/2 шт. Паттерн: generate → `job_status` (sync) → `curl` rawUrl → `./assets/`.
- **Tier 1**: Pexels/Unsplash API если есть ключ в env.
- **ПРАВИЛО САМОДОСТАТОЧНОСТИ:** любой прототип с картинкой обязан выглядеть законченным, **если картинка не загрузилась** — никаких пустых панелей-заглушек. Под фото — фон в той же палитре; фото как усиление, не как несущая конструкция. (Главный урок теста: hotlink-дыры = «позорище».)
- Минимум 2 из 6 прототипов используют реальные изображения (hero/фон/секции).

**3f. Мотион — переосмыслить (НОВОЕ).**
Заменить механический «particle system mandate» на 4 дефолта: Ken Burns, reveal-on-scroll (+фолбэк), animated gradient, scroll-progress. Плюс View Transitions для секций. Старый animation-guide (particles/lightning) → оставить как «cinematic tier» для 1 прототипа, но не как обязаловку.

**3g. Scope.** 4/6 прототипов = hero. **1-2 прототипа = 2-3 секции scroll-нарратива** (где и живёт «вау»).

**3h. Free variant — переосмыслить.**
Вместо «polished and distinctive» (→ среднее) дать **creative challenge**: «Выбери одну визуальную идею, которую ты НЕ ожидал бы увидеть в этой категории продукта, и построй весь прототип вокруг неё. Рискни.»

**3i. Сохранить:** grayscale-тест, параллельную генерацию субагентами, mobile-first, desktop utilization rules, performance rules, Cyrillic safety.

### Step 3.5 — WOW-гейт перед записью (НОВОЕ)
Дополнить 13 технических пунктов творческими гейтами:
- «Увидел бы незнакомец это без контекста — почувствовал бы что-то сильное?»
- «Есть здесь хоть один ход, которого ты раньше не видел?»
- «Если убрать цвет — отличается от других? (grayscale-тест)»
- «Это выглядит как СДЕЛАННОЕ или как СГЕНЕРИРОВАННОЕ?»
Если честный ответ «нет» — концепция слабая, переделать.

### Step 4 — Present & Evaluate — *усилить + Playwright*
- Добавить **опциональный Playwright-шаг**: скриншот каждого прототипа → PNG → настоящая визуальная галерея (чинит iframe-на-file:// боль).
- Добавить в evaluation вопрос: «Какой из них ты мог бы представить как полностраничную рекламу в дизайн-журнале?»
- Оставить mixing-вопрос.

### Step 5 — Refine & Build — *оставить*
Поднять «Design quality note» (не используй генераторы палитр) из футноута в принцип Step 3.

## Что НЕ трогаем (хорошее, проверенное)
Контекст-интервью, Design DNA, grayscale-тест, BRIEF.md чекпоинт, параллельная генерация, gallery, mixing-вопрос, mobile-first/desktop utilization, performance/accessibility rules, Cyrillic safety.

## MCP-решения
- **НЕ** интегрируем 21st/shadcn/Figma (React-only, generic, ломают standalone).
- **Playwright MCP** — опциональный шаг галереи.
- **Higgsfield `generate_image`** — Tier 2 изображений (опционально, если доступен).

## Риски редизайна
- Скилл может стать слишком длинным → выносить большие блоки (frontend_aesthetics, image-recipes, motion-snippets, art-direction-примеры) в `references/`.
- «Рискуй» без вкуса → грязь. Баланс: концепция + DNA рефов + WOW-гейт держат планку.
- Tier 2 (генерация картинок) медленный/платный → дефолт Tier 0, апгрейд по запросу.

## План теста (до написания финала)
Сгенерировать на одном живом брифе **2 прототипа одного концепта**:
1. «Старый» способ (free variant в текущем стиле — polished/safe).
2. «Новый» способ (art-directed: концепция + frontend_aesthetics + отличительный шрифт + реальное изображение + Ken Burns/scroll-reveal).

Сравнить визуально (через браузер/скриншот). Если дельта «вау» очевидна — методология валидна, пишем финальный SKILL.md. Если нет — правим план.
