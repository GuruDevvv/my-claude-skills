Промпт: Полный аудит организации проекта
Открой новую сессию Claude Code в папке проекта и вставь всё ниже разделителя как сообщение в чат.

Проведи полный аудит организации этого проекта. Весь отчёт на русском языке. Действуй строго по порядку.

0. Проверка окружения
Перед началом определи что есть:

Git:

Выполни git status. Если не git-репо — запомни, шаг Git-гигиены пропустишь.
CLAUDE.md:

Проверь наличие CLAUDE.md в корне проекта и в подпапках. Запомни где есть, где нет.
Memory:

Memory Claude Code хранит НЕ в папке проекта, а в ~/.claude/projects/<slug>/memory/.
Чтобы найти правильный slug, выполни: ls ~/.claude/projects/ | grep -i "<имя проекта>"
Проверь существование этой папки и MEMORY.md в ней.
Также проверь, есть ли memory/ внутри самого проекта (бывает и так).
Скиллы:

На шагах 1 и 2 попробуй вызвать скиллы:
claude-md-management:claude-md-improver — аудит и улучшение CLAUDE.md файлов (проверяет структуру, полноту, актуальность, предлагает улучшения)
claude-code-setup:claude-automation-recommender — анализ проекта и рекомендации по автоматизациям Claude Code (хуки, MCP серверы, скиллы, настройки)
Если скилл сработал — используй его результат.
Если скилл НЕ найден — сообщи: «Скилл [название] не найден. Он нужен для [описание]. Хочешь инструкцию по установке?» И выполни ручной фолбэк (указан в каждом шаге).
Размер проекта:

Посчитай примерное количество файлов (без .git и node_modules).
Если больше 200 — спроси: «В проекте ~N файлов. Провести полный аудит или сфокусироваться на корне, CLAUDE.md и memory?»
Покажи краткую сводку окружения перед тем как продолжить:

Окружение:
- Git: ✅/❌
- CLAUDE.md: [где найден]
- Memory: [путь или ❌]
- Файлов в проекте: ~N
⛔ ПРАВИЛА БЕЗОПАСНОСТИ (действуют на ВСЕХ шагах)
НЕ читай содержимое файлов .env, credentials*, *.pem, *.key, *secret*
НЕ выводи значения API ключей, токенов, паролей — даже если увидел их в CLAUDE.md или MEMORY.md. Пиши: [СЕКРЕТ скрыт]
НЕ исправляй ничего автоматически — только отчёт
Ограничивай вывод команд: добавляй | head -30 к любой команде, которая может дать больше 30 строк
Если файл бинарный (PDF, DOCX, XLSX, изображение) — не читай, только отметь имя и размер
1. CLAUDE.md аудит
Вызови скилл claude-md-management:claude-md-improver. Если не найден — выполни ручной фолбэк ниже.

Ручной фолбэк: Проверь каждый найденный CLAUDE.md:

Есть описание проекта?
Структура папок соответствует реальной?
Все ссылки на файлы валидны? (проверь что файлы существуют по указанным путям)
Нет устаревших дат, статусов, URL?
Нет дублирования с MEMORY.md?
Нет пустых/заглушечных секций?
Нет чувствительных данных (ключи, токены)?
2. Рекомендации по автоматизациям Claude Code
Вызови скилл claude-code-setup:claude-automation-recommender. Если не найден — выполни ручной фолбэк ниже.

Ручной фолбэк: Оцени:

Есть ли хуки (pre-commit, lint, format)? Нужны ли они для этого типа проекта?
Подключены ли MCP серверы? Какие были бы полезны?
Есть ли .claude/settings.local.json? Нет ли в нём устаревших путей или мёртвых записей?
Тип проекта (код / документация / смешанный) — соответствуют ли текущие настройки типу?
Если проект не кодовый (документация, контент) — НЕ рекомендуй линтеры/форматтеры/тесты
3. Структура файлов
Покажи дерево папок (максимум 2 уровня, | head -30)
Найди файлы в корне, которые НЕ являются стандартными (README, CLAUDE.md, CHECKLIST, package.json, .gitignore, конфиги). Обрати внимание на: данные, HTML, изображения, PDF, документацию — им место в подпапках.
Найди пустые папки, дубликаты, временные файлы (.tmp, .bak, копии с цифрами)
Найди большие файлы (>1MB), исключая .git и node_modules. Используй Glob и Bash.
Проверь .gitignore — покрыты ли: node_modules/, .env*, бинарные форматы? Нужны ли дополнения?
4. Memory аудит
Пропусти если memory не найдена в шаге 0.

Для каждой найденной memory-папки (проверяй только файлы первого уровня ссылок из MEMORY.md, НЕ переходи по ссылкам внутри этих файлов):

Прочитай MEMORY.md (только текст, НЕ переходи по ссылкам на бинарные файлы)
Все ли ссылки из MEMORY.md ведут на существующие файлы?
Есть ли файлы в папке memory/, на которые MEMORY.md НЕ ссылается?
Есть ли дублирование между MEMORY.md и CLAUDE.md? (одна и та же информация в двух местах)
Есть ли чувствительные данные (ключи, токены)? → отметь как [СЕКРЕТ], не выводи значение
Критерий устаревшего: ссылается на несуществующие файлы, упоминает удалённые URL/деплои, или содержит даты/статусы которые явно устарели (например статус «В процессе» для задач полугодовой давности)
5. Git гигиена
Пропусти если не git-репо (определено в шаге 0).

Если коммитов нет (git log --oneline -1 пуст) — отметь это в отчёте и пропусти проверки tracked файлов.
git status | head -30 — покажи untracked файлы
Проверь tracked файлы на бинарники: git ls-files | grep -iE '\.(jpg|jpeg|png|gif|pdf|docx|xlsx|pptx|zip|rar|mp4|mp3)$' | head -20
Если бинарники найдены — покажи суммарный размер
Проверь tracked файлы на секреты: git ls-files | grep -iE '^\.env|/\.env|credentials\.|\.pem$|\.key$' | head -10
НЕ дублируй находки из шага 3 — ссылайся: «см. шаг 3»
6. Финальный отчёт
Собери ВСЁ в единый отчёт. Включи статистику покрытия.

Просмотрено файлов: ~X | Прочитано детально: Y | Найдено проблем: Z
⏭️ Не проверялось
(что пропущено и почему — скиллы не установлены, нет git, нет memory и т.д.)
✅ Всё ОК
(что проверено и в порядке)
⚠️ Рекомендации (порядок, оптимизация, удобство)
(что стоит улучшить, не критично)
🔴 Требует действий (безопасность, поломки, битые ссылки)
(что нужно исправить)
Критерии:

🔴 = секреты в открытом виде, битые ссылки в CLAUDE.md, файлы >10MB в git, отсутствие .gitignore для чувствительного
⚠️ = дублирование, устаревшие записи, бинарники в git, неоптимальная структура
📋 План действий
(нумерованный список, сгруппированный: сначала 🔴, потом ⚠️)

Жду твоего решения — какие пункты выполнить? Когда выберешь — выполню только выбранные, по одному, с подтверждением каждого шага.
