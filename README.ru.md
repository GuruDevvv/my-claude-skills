# multi-layer-review

[🇬🇧 Read in English](README.md)

Скилл для Claude Code, который прогоняет спек или ТЗ через цепочку до 5 ревьюеров последовательно — до начала разработки. Каждый ревьюер передаёт находки следующему: тот либо подтверждает, либо опровергает, либо добавляет новое.

---

## Как это работает

**Шаг 0 — Ambiguity Gate (автоматически).**
Перед запуском ревьюеров скилл проверяет спек на неоднозначности: размытые требования («быстро», «удобно»), противоречия между секциями, отсутствующие ограничения. Если находит — спрашивает уточнить или продолжить.

**Шаги 1–5 — Ревьюеры.**
Каждый смотрит на спек со своей точки зрения. Начиная со второго, каждый видит находки предыдущих — и либо подтверждает, либо опровергает, либо добавляет новое. Это исключает дублирование и усиливает критику.

**Финал — Синтез.**
Все находки объединяются, дедуплицируются и сортируются по приоритету: Critical / Important / Minor.

---

## Ревьюеры

| # | Роль | Что ищет |
|---|------|----------|
| 1 | **Plan Agent** — архитектор | Структурные пробелы, качество API design, граничные случаи, противоречия между секциями |
| 2 | **Code Reviewer** — разработчик | Риски реализации, небезопасные паттерны, проблемы зависимостей, отсутствующая обработка ошибок |
| 3 | **Devil's Advocate** — скептичный пользователь | «Что если...» — UX-провалы, неожиданное поведение, что сломается в первый же день |
| 4 | **Robustness Checklist** — системный инженер | Оценка 0–3 по 8 критериям: error handling, concurrency, безопасность, тестируемость и др. |
| 5 | **Requirements Traceability** — аналитик | Правильную ли задачу решаем? Соответствует ли решение бизнес-цели? Есть ли критерии успеха? |

Ревьюеров можно выбирать — не обязательно запускать всех пятерых.

---

## Когда использовать

- Написал спек или ТЗ — хочешь найти проблемы до начала разработки
- Проектируешь API или архитектуру модуля
- Не уверен что правильно понял задачу или решаешь ли ту проблему

---

## Пример

```
Ты: проверь ТЗ
```

Скилл спросит каких ревьюеров запустить, попросит прислать спек (файлом или текстом), прогонит через выбранных и выдаст синтез с указанием кто что нашёл. Результат сохраняется в файл.

## Установка

Посмотри [что устанавливается](multi-layer-review/SKILL.md) перед запуском команды.

**macOS / Linux:**
```bash
mkdir -p ~/.claude/skills/multi-layer-review
curl --fail -o ~/.claude/skills/multi-layer-review/SKILL.md \
  https://raw.githubusercontent.com/GuruDevvv/my-claude-skills/main/multi-layer-review/SKILL.md
```

**Windows (Git Bash или WSL):**
```bash
mkdir -p "$USERPROFILE/.claude/skills/multi-layer-review"
curl --fail -o "$USERPROFILE/.claude/skills/multi-layer-review/SKILL.md" \
  https://raw.githubusercontent.com/GuruDevvv/my-claude-skills/main/multi-layer-review/SKILL.md
```

Перезапусти сессию Claude Code — терминал с `claude`, или перезагрузи окно VS Code.

**Проверка:** напиши агенту `multi-layer review` — скилл активируется автоматически.
